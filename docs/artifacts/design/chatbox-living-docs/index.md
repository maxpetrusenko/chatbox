## Goal

Define how living-docs artifacts should look and behave in this repo so humans skim fast and agents still get exact state through sidecars.

## Required Sections

- artifact identity
- freshness and status
- current repo truth
- next action
- drift or risk summary

## Visual Rules

- strong serif title
- compact eyebrow labels
- warm paper background
- explicit callout cards
- one main accent and one support accent

## Chatbox Rule

If a feature has hidden runtime, role, auth, or heuristic gates, the visual artifact must show them above the fold.
