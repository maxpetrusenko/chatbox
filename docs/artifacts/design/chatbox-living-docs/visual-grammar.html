Visual Grammar · Human Skim

Human skim first.
Machine detail second.

This grammar defines page order. Approval starts with a fast visual read, then markdown and sidecars carry the exact detail.

First Screen

Must show first

1

what this artifact is

2

canonical or source seed

3

freshness and status

4

next action

Never hide

Trust-critical content

  • repo truth
  • hidden gates
  • active drift
Page rhythm

Recommended layout

  • hero with approval headline
  • small stats or phase counters
  • 2 to 4 proof cards
  • one explicit next-action callout
Approval test

What a human should answer fast

What is this, how fresh is it, what repo truth matters, what is missing, and what do we do next. If the page does not answer those in one screen, it failed the grammar.
