How To Use This Hub

Read by decision, not by folder

  • use plugin docs when deciding what the shipped plugin platform really does
  • use runtime/storage docs when debugging session or persistence behavior
  • use requirements and cost docs as historical context, not sole product truth
  • treat older provider and ops guides as needing command sanity-checks against the current repo scripts
Why This Matters

What this page prevents

Without this split, a skim reader can easily mistake historical briefs, language-specific notes, and current implementation references for the same class of truth. This hub makes that distinction explicit.
